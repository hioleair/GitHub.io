如何使用hexo搭建独立静态博客

一.前言

上篇《如何利用github免费快速搭建个人静态博客》已经说完了安装和启动，这篇我们详细讲解一下如何部署服务和发布内容。

二.注册github账号

​ 如果你已经注册过github可以跳过这一步。

​ 访问github网站：http://github.com 这里简单介绍一下github网站，git是linux内核开发者，就是我们俗称的linux之父开发的用于管理分布式版本的控制系统，github就是支持且只支持git进行版本库托管，然后面向开源或私有项目的托管平台，任何人都可以注册账号发布你的开源代码在github免费托管，同时你也可以查到各种优秀的开源项目在github。

​ 好，了解了github以后，我们开始github的注册使用之旅，因为是国外的网站，所以github都是英文，不过不用担心我这边一步一步详细的教大家如何注册。

打开网页是这样的，然后username填写自己的用户名，不可以和别人重名，email填写自己有效的邮件地址，然后是password密码，如果内容都是正确的后面会有绿色的小勾，然后点击sign up for github即可。

注册出来是这样子的，然后这些信息填不填写都没差，我们直接选择continue

这里我们直接跳过选择skip this step就行

然后就是这样了

这时候需要登录你刚刚填写的邮箱进行验证了，打开github给你发的链接，然后输入用户名和密码点击sign in登录即可，这样你就算是正式完成了注册。(我是手机登录好的邮箱就懒得在电脑登录重新邮箱了，电脑登录的邮箱操作一样，打开github发你链接，登录你注册github的登录名和密码)

你重新再电脑端登录进来就可以开始新建自己的仓库就是new repository了，大概就是点击你的头像旁边的+号，选择new respository。

然后给你仓库起名字，随便叫什么都行，不过如果你不熟悉github并且这是搭建博客用的最好还是和我这样叫github.io吧，然后create repository就可以创建成功了，接下来我们只要把本地的hexo博客发布到我们仓库，然后很快你就能在网页上访问到自己的博客了

成功create repository以后，会进到这个界面，我们选择http点击复制。

​ 然后新建一个text文件保存这个地址，接下来我们部署服务会用到的。

三.部署发布服务

1.本地hexo配置更改

​ 部署之前，我们找到hexo的配置文件进行配置，就是在我们放在E盘的实例化的hexo文件夹，文件结构如下，选择_config.yml，（扩展小知识：这里给大家介绍一下.yml格式的文件是目前比较常见流行的配置文件格式，spring boot也一般都是用.yml格式的配置文件，它比.xml的配置文件要简洁很多，它一般就是像这样 主键: 配置信息，如果带有子配置信息就会在tab键对齐的下面又带一对键值对，这里的配置信息前面一定要带有空格，不然就会报错，因为以后自定义hexo皮肤主题和其他配置都会更改配置文件，所以特别强调一下，这次我们只有注意一下就行）

2.hexo发布命令

这个时候，我们已经完成了部署任务的百分之九十九了，是不是很简单，现在我们只需要简单俩条条命令就能完成部署，下面就是见证奇迹的时候，

第一条命令就是npm install hexo-deployer-git –save

这条命令要在cmd命令框下运行，cmd下面才能调出npm的环境变量，在git bash会失败，请注意

第二条命令就是hexo clean&&hexo g&&hexo d

这条命令要在git bash下运行，因为要用到git，运行出来会要你输入用户名和密码，用户名就是你注册github的名字，密码就是你的登录密码，等出现INFO deploy done：git提示的时候说明部署成功

我来大概解释一下命令哈，第一条就是hexo配套的发布工具，npm就是包管理工具命令，我们这条命令就是安装hexo-deployer-git，有了它hexo d部署的时候才不会出错。

第二条命令就是&&是把三个命令同时联结一起执行的意思，所以其实第二条是走了三个步骤，hexo clean 清理缓存，hexo g重新生成静态文件，hexo d发布。

3.github的最后配置

​ 然后我们登录github，进行最后的配置，找到自己配置的仓库，然后点击setting在设置里面GitHub Pages选择对source进行配置选择master branch等待网页刷新即可看到你生成的个人博客了

四.购买域名并配置

​ 这一步不是必须的一步了，因为其实你的网站已经用了，不过你的网站地址目前是github给你分配的二级域名，如果你觉得二级域名不够高级，你可以自己买一个域名，然后再配置DNS绑定到你的地址就可以了。
